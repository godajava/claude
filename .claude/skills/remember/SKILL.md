# 기억하기 (일정/할일/메모 기록)

사용자의 자연어 입력을 분석해 일정, 할 일, 메모 중 하나로 분류하고 해당 파일에 기록합니다.

## 분류 기준

| 유형 | 신호 | 기록 위치 |
|------|------|-----------|
| **일정** | 특정 날짜+시간의 약속·회의·이벤트 ("내일 3시에 회의") | `secretary/schedule.md` |
| **할 일** | 해야 하는 행동, 마감이 있거나 완료 가능한 작업 ("보고서 제출해야 해") | `secretary/todo.md` |
| **메모** | 기억할 정보, 아이디어, 사실 ("와이파이 비밀번호는 ...") | `secretary/notes/YYYY-MM-DD-<주제>.md` |

애매하면: 시간이 정해져 있으면 일정, 행동이 필요하면 할 일, 둘 다 아니면 메모로 분류합니다. 분류 결과를 사용자에게 알려주고, 잘못됐다면 옮겨 기록합니다.

## 절차

1. KST 기준 오늘 날짜를 확인합니다: `TZ=Asia/Seoul date +%Y-%m-%d`
2. 상대적 날짜 표현("내일", "다음 주 금요일", "이번 달 말")을 구체적 날짜로 환산합니다.
3. 유형별 형식에 맞춰 기록합니다:
   - **일정**: `secretary/schedule.md`의 해당 날짜 섹션(`## YYYY-MM-DD (요일)`)에 시간순으로 삽입. 섹션이 없으면 날짜순 위치에 새로 만듭니다. 형식: `- HH:MM 제목 — 장소 (메모)`
   - **할 일**: `secretary/todo.md`에 추가. 형식: `- [ ] 내용 (마감: YYYY-MM-DD) #태그` (마감 없으면 생략)
   - **메모**: `secretary/notes/YYYY-MM-DD-<주제>.md` 생성 또는 같은 주제 파일에 추가. 상단에 `# 제목`과 작성일 기재.
4. 기록 후 한 줄로 확인해 줍니다: 예) "✅ 일정으로 기록했습니다: 2026-07-10 15:00 팀 회의 (schedule.md)"

## 규칙

- 파일이나 디렉토리가 없으면 만들고 진행합니다.
- 하나의 입력에 여러 항목이 섞여 있으면(일정+할 일 등) 각각 분리해 모두 기록합니다.
- 기존 항목과 중복이면 기록하지 않고 이미 있다고 알려줍니다.
- 일정 충돌(같은 시간대 겹침)을 발견하면 기록은 하되 충돌 사실을 알려줍니다.
